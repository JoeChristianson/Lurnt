# Lurnt — Product Spec

## Vision

Lurnt is an AI-powered learning platform that guides users from beginner to expert in any subject. It builds a personalized knowledge graph for each expertise, progressively expanding it as the user advances, and uses LLM-driven assessments to verify understanding.

## Core User Flow

### 1. Choose an Expertise

A user declares what they want to learn (e.g., "Machine Learning"). They can have multiple active expertises simultaneously.

### 2. Initial Assessment

- **Conversational intake:** An LLM conversation explores the user's existing background, experience, and goals.
- **Placement test:** Based on the intake, the system generates a quiz to verify the user's claimed knowledge level and determine which nodes to mark as already completed.

### 3. Graph Generation

The LLM generates an initial set of knowledge graph nodes based on the assessment results. Each node represents a concept or skill (e.g., "Matrix Multiplication", "Gradient Descent", "Backpropagation").

Nodes are connected by prerequisite edges forming a DAG — you can't attempt "Backpropagation" until you've completed "Partial Derivatives" and "Chain Rule".

The graph is **not** generated all at once. It expands progressively as the user advances, with the LLM adding new nodes ahead of the user's frontier.

### 4. Active Learning

The system always presents **at least three unlocked topics** — nodes whose prerequisites are all completed. The user picks one and:

1. **Studies resources** — Each node has curated learning resources (videos, articles, courses, exercises). Resource types are filtered by the user's stated preferences.
2. **Takes the quiz** — A hard-coded set of questions generated by the LLM when the node was created. Tests factual recall and basic understanding.
3. **Conversational assessment** — If the quiz is passed, an LLM conducts a deeper conversational assessment to verify genuine understanding. The LLM probes edge cases, asks for explanations, and checks for conceptual gaps.
4. **Node complete** — On passing both, the node is marked complete. New downstream nodes become available.

### 5. Ongoing Expansion

There is no "completion" state for an expertise. The graph is always expandable — the LLM can generate deeper, more specialized nodes as the user progresses into advanced territory.

## Resource System

### Discovery
When a node is first encountered by any user, the LLM looks up relevant learning resources (YouTube videos, articles, documentation, courses, etc.) and stores them.

### Caching
Resources are persisted and reused for subsequent users who reach the same node. This avoids redundant LLM lookups.

### Community Curation
- Users can **submit** additional resources for any node.
- Users can **upvote** resources to surface the best ones.

### Personalization
Users set preferences for resource types (e.g., "prefer videos over articles", "include textbook references"). The system filters and ranks resources accordingly.

## Domain Model (High Level)

- **User** — Account, auth, preferences (resource types), multiple active expertises.
- **Expertise** — A subject area (e.g., "Machine Learning"). Links a user to a knowledge graph instance.
- **KnowledgeGraph** — A collection of nodes and edges for a specific expertise. May be shared/templated across users or personalized.
- **Node** — A single concept/skill. Contains: title, description, quiz questions, assessment prompts, status (locked/unlocked/completed).
- **Edge** — A prerequisite relationship between two nodes (directed, acyclic).
- **Resource** — A learning resource linked to a node. Has: URL, type (video/article/course/etc.), source, upvote count, submitted-by.
- **Assessment** — Record of a user's quiz and conversational assessment attempts on a node.

## Open Questions

- How to handle conflicting knowledge graphs when the same expertise is generated for different users? Share a canonical graph and personalize the starting point, or fully independent graphs per user?
- Spaced repetition / review — should completed nodes decay over time and require re-assessment?
- How deep should the conversational assessment go? Fixed number of exchanges, or adaptive?
- Should the system track time spent studying resources before allowing quiz attempts?
- Rate limiting on LLM calls — how to handle cost at scale?
